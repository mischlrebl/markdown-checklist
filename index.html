Markdown Checklist

Paste or edit markdown on the left — the checklist renders live on the right.

Click a checkbox to cycle: unchecked → checked → not applicable (N/A, skip).
Click the progress stats next to a section heading to change all items in that section.

Ctrl+Z undoes the last change — a checkbox toggle, a burst of typing, Format, or Clear/Load demo/Check all.

Everything is auto-saved in the browser and survives refresh and browser restarts.

For maximum focus, hide the editor with Editor, then switch to Focus mode — only the checklist and progress bar remain.

Each line in the editor becomes a separate element in the checklist — unlike standard markdown, a single line break is enough; no blank line needed. This also applies to blockquotes.
N/A (- [-]) is not standard markdown.

Use Export to save the checklist as markdown — states are written as - [ ] / - [x] / - [-] and a progress summary is prepended as a blockquote — paste the file back to restore the exact state.

A checklist has two lives: the template — the blank, reusable structure you open with Open and keep in version control — and the run — the filled-out result you capture with Export after completing a process. Keep them separate: the template stays clean for the next run; the exported result documents what happened.

Use Syntax in the editor panel for the full reference and a built-in example.
